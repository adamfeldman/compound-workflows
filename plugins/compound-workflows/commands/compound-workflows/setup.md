# Compound Workflows Setup

Detect what's available in the current environment and configure the project for compound workflows. Run this after installing the plugin, or anytime to check your setup.

## Phase 1: Detect Environment

Run all detection checks in parallel:

```bash
# Check for beads
bd version 2>/dev/null && echo "BEADS=installed" || echo "BEADS=missing"

# Check for GitHub CLI
gh version 2>/dev/null && echo "GH=installed" || echo "GH=missing"

# Check for compound-engineering plugin (look for a known agent)
ls ~/.claude/plugins/cache/*/compound-engineering/*/agents/review/security-sentinel.md 2>/dev/null && echo "CE=installed" || echo "CE=missing"
```

Check for PAL MCP by attempting to list models:

```
Try: mcp__pal__listmodels
If it returns models → PAL=installed
If tool not found → PAL=missing
```

## Phase 2: Present Status

Display the results as a status table using **AskUserQuestion**:

Build the status message dynamically based on detection results. For each enhancement, show its status and what it unlocks:

**Present this information to the user:**

```
## Environment Status

| Enhancement | Status | What it unlocks |
|-------------|--------|-----------------|
| **beads** (`bd`) | [Installed/Not found] | Compaction-safe task tracking in /compound-workflows:work and /compound-workflows:work-agents. Without it: TodoWrite fallback (functional but loses state on compaction) |
| **PAL MCP** | [Connected/Not found] | Cross-model red-team challenges in /compound-workflows:brainstorm and /compound-workflows:deepen-plan (Gemini, GPT, etc. catch blind spots Claude misses). Without it: Claude subagent fallback (still works, single-model) |
| **compound-engineering** | [Installed/Not found] | Specialized review/research agents (security-sentinel, performance-oracle, etc.) with rich domain prompts. Without it: general-purpose fallback (still works, less specialized) |
| **GitHub CLI** (`gh`) | [Installed/Not found] | PR creation in /compound-workflows:work and /compound-workflows:review. Without it: manual PR creation |
```

Use **AskUserQuestion**:

```
question: "Environment detected. What would you like to do?"
header: "Setup"
options:
  - label: "Continue setup"
    description: "Configure project directories and save settings"
  - label: "Install missing enhancements"
    description: "Show installation instructions for missing tools"
  - label: "Done"
    description: "Environment looks good, skip further setup"
```

### If "Install missing enhancements"

Show installation instructions for each missing enhancement:

**beads:**
```
# Install beads marketplace and plugin
# See: https://github.com/steveyegge/beads
claude /install beads
```

**PAL MCP:**
```
# Add PAL to your MCP server configuration in ~/.claude/settings.json or project .claude/settings.json
# PAL provides multi-model AI access for red-team challenges
# See PAL documentation for setup instructions
```

**compound-engineering:**
```
# Install compound-engineering for specialized review agents
claude /install compound-engineering
```

**GitHub CLI:**
```
# macOS
brew install gh
gh auth login

# Other platforms: https://cli.github.com/
```

After showing instructions, return to Phase 2 to re-detect.

### If "Done"

Skip to Phase 4.

## Phase 3: Configure Project Directories

Check which compound workflow directories exist:

```bash
for dir in docs/brainstorms docs/plans docs/solutions docs/decisions .workflows; do
  if [ -d "$dir" ]; then
    echo "EXISTS: $dir"
  else
    echo "MISSING: $dir"
  fi
done
```

If any directories are missing, use **AskUserQuestion**:

```
question: "The following directories are used by compound workflows but don't exist yet: [list]. Create them?"
header: "Directories"
options:
  - label: "Create all (Recommended)"
    description: "Create all missing directories for the full workflow cycle"
  - label: "Skip"
    description: "I'll create them manually as needed"
```

If "Create all":
```bash
mkdir -p docs/brainstorms docs/plans docs/solutions docs/decisions .workflows
```

## Phase 4: Write Local Config

Write `compound-workflows.local.md` to the project root with detected capabilities:

```markdown
---

# Re-run setup to update

tracker: [beads|todowrite]
red_team: [pal|claude-subagent]
review_agents: [compound-engineering|general-purpose]
gh_cli: [true|false]
---

# Compound Workflows Configuration

## Environment
- **Task tracking:** [beads (compaction-safe) | TodoWrite (in-memory only)]
- **Red team:** [PAL MCP (multi-model) | Claude subagent (single-model)]

- "Performance-critical: we serve 10k req/s"
```
